04 · Key Concepts
🔢
Tokens
The atomic units LLMs process. Roughly ¾ of a word on average. GPT-4 has a vocabulary of ~100k tokens. The model only ever sees token IDs — never raw text.
🎯
Attention
The core transformer mechanism. Every token can "attend" to every other token in context, letting the model capture long-range relationships and meaning.
📏
Context Window
The max tokens the model can see at once. GPT-4 supports 128k. The model has no memory beyond the window — the full conversation history must fit inside it.
🌡️
Temperature
Controls randomness. Near 0 = always pick the most likely token (deterministic). Near 2 = nearly flat distribution, highly unpredictable output.
🏋️
Parameters
The weights learned during training. GPT-4 has an estimated ~1.8 trillion. These encode all the model's "knowledge" and are frozen at inference time.
📋
System Prompt
Instructions prepended before the conversation. Sets persona, constraints, and behavior. Invisible to the user but part of the model's full context on every turn.
🔄
Autoregression
The model generates one token at a time. Each sampled token is appended to the context, and the model runs again. This loop continues until an end-of-sequence token.
📐
Embeddings
Each token ID maps to a vector of thousands of numbers. These high-dimensional vectors are what actually flow through the transformer — not the token text.
🎲
Softmax
Converts the model's raw output scores (logits) into a valid probability distribution that sums to 1.0. Every token in the vocabulary gets a probability, however tiny.
