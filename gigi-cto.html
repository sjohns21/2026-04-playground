Graph System Blueprint (V1)

  • Storage: Postgres + vector index + adjacency table, then graduate hot graph ops into dedicated graph store as load warrants.
  • Ingestion: Event-driven pipeline for profile updates, interactions, and intro outcomes with idempotent upserts.
  • Ranking: Hybrid retrieval (explicit edges + embeddings) followed by feature-scored reranker with trust constraints.
  • Safety: Hard double-opt-in enforcement in orchestration layer plus abuse/rate control.

Unstructured Data Strategy

  • Normalize profile text, context snippets, intent statements, and conversation metadata into typed feature records.
  • Build a feature store for recency, relevance, and intro-success priors.
  • Use embedding + lexical hybrid retrieval to keep cold-start and long-tail recall high.
  • Instrument everything: retrieval precision, intro acceptance rate, response latency, trust score drift.

// candidate matching sketch: graph + embeddings + trust constraints
function rankCandidates(goal, userId) {
  const neighborhood = graph.expand(userId, { hops: 2, minEdgeWeight: 0.35 });
  const lexical = search.keyword(goal.text, { within: neighborhood.ids });
  const semantic = search.vector(embed(goal.text), { within: neighborhood.ids });

  const merged = mergeAndDedup(lexical, semantic);
  const filtered = merged.filter(c => c.introConsent && c.blocked !== true);

  return rerank(filtered, {
    trustAffinity: scoreTrust(userId, c.id),
    goalFit: scoreGoal(goal, c.profile, c.activity),
    introProbability: model.pAccept(userId, c.id)
  }).slice(0, 5);
}
